# 網址 Web Address(URL)

統一資源定位符（英語：Uniform Resource Locator，縮寫：URL，或稱統一資源定位器、定位位址、URL位址[1]）俗稱網頁位址，簡稱網址，是網際網路上標準的資源的位址（Address），如同在網路上的門牌。

# HTTP通訊協定是網站前後端互動的基礎

# Client & Server

一般來說傳輸資料的兩端會分為 客戶端 ( Client ) 跟 伺服器端 ( Server )

  • Client： 以網頁來說就是你的瀏覽器、電腦，主要會發送 「 請求 request 」到 Server 端

  • Server： 收到 request 開始處理資料，完成後會回傳 「回應 response 」到 Client 端

  • 在瀏覽器中輸入一串網址: http://www.google.com

  • 按下 Enter

  • 其中, (www.google.com) 實際上會對應到一個實體的IP地址, 例如(142.251.42.4), 點擊 Enter 後, 我們的瀏覽器就會發送一個請求到 Server端 的實體位置, 告訴對方:"我到了你的首頁位置哦, 請把你的首頁檔案回傳給我", 該電腦(伺服器) 接收到我們的請求後, 就會將它首頁的檔案回傳給我們, 這個檔案經過瀏覽器的解析後, 就會顯示出我們看到的網頁畫面

    • 當然不止在瀏覽首頁時會發送數據的請求, 很多瀏覽器的交互事件也會使我們的瀏覽器向 Server端 發送請求, 例如: 在購物網站裡, 當我們點擊某商品的購買按鈕時, 我們的瀏覽器就會發送請求給 Server端, 告訴對方:"我們這裡需要新增一個訂單"
      • 注意: 雖然點擊購買時, 瀏覽器的網址可能並沒有發生變化, 但實際上已經通過程式碼在看不見的地方向不同的網址發送了請求

# HTTP Method

根據發送請求的目的不同, 分了不同的幾種方法, 例如:
  • GET 拿取資料
  • POST 上傳資料
  • PUT 覆蓋資料
  • PATCH 更新資料
  • DELETE 刪除資料

# HTTP Status Code

1xx: informational 
(接收到request了, 需要繼續處理)

2xx: Successful
(請求成功)
常見: 
    - 200 - OK  (ok沒問題)
    - 201 - Created  (創建好了, 沒問題)

3xx: Redirects
(重新定向 轉址)
常見: 
    - 301 - Moved Permanently  (你請求的網址已經搬家了)


4xx: Client Errors
(客戶端錯誤)
常見: 
    - 400 - Bad Request  (你的請求格式可能寫錯了)
    - 401 - Unauthorized  (你請求的資源較敏感, 需要經過驗證, 但你沒有攜帶驗證碼)
    - 403 - Forbidden  (你請求的資源較敏感, 你有攜帶驗證碼, 經過驗證後, 你的驗證碼不符合資格)
    - 401 - Not Found  (你發送請求的網址不存在)


5xx: Server Errors
(伺服器端錯誤)
常見: 
    - 500 - Internal Server Error  (伺服器錯誤)

不安全

HTTP協定是不安全的, 因為HTTP傳送資料的方式是明文傳輸

如果你在購物網站通過HTTP協定發送請求, 那麼你購買的商品資訊, 你的付款資訊, 個人資訊等等都是明文傳輸給Server端的, 這樣就很容易被有心人士竊聽, 盜取你的個人資訊,

# HTTPS協定

超文本傳輸安全協定（英語：HyperText Transfer Protocol Secure，縮寫：HTTPS；常稱為HTTP over TLS、HTTP over SSL或HTTP Secure）是一種透過計算機網路進行安全通訊的傳輸協定。HTTPS經由HTTP進行通訊，但利用SSL/TLS來加密封包。HTTPS開發的主要目的，是提供對網站伺服器的身分認證，保護交換資料的隱私與完整性。這個協定由網景公司（Netscape）在1994年首次提出，隨後擴展到網際網路上。

source: wikipedia
