# Typechecking With PropTypes (類型檢查)

隨著你的應用程式不斷增長，你可以通過類型檢查捕獲大量錯誤。對於某些應用程式來說，你可以使用 Flow 或 TypeScript 等 JavaScript 擴展來對整個應用程式做類型檢查。但即使你不使用這些擴展，React 也內置了一些類型檢查的功能。要在元件的 props 上進行類型檢查，你只需配置特定的 propTypes 屬性：

  • 第一步: 引入
https://www.npmjs.com/package/prop-types
1
  • 使用prop傳遞數據:




 








 










class Child extends React.Component{
    render(){
        return(
            <div>
                name: {this.props.name}
            </div>
        )
    }
}
class Parent extends React.Component{
    render(){
        return(
            <div>
                <Child name = {999} />
            </div>
        )
    }
}

let root = ReactDOM.createRoot(document.getElementById('root'))
root.render(<Parent />)

// 頁面的呈現效果為: name: 999

  • 添加類型判斷

 






Child.propTypes = {
    name: PropTypes.string // 限定 name屬性 的值類型為 string類型
}

// 此時控制台內會報錯: Warning: Failed prop type: Invalid prop `name` of type `number` supplied to `Child`, expected `string`. 
// PropTypes 提供一系列驗證器，可用於確保元件接收到的數據類型是有效的。在本例中, 我們使用了 PropTypes.string。當傳入的 prop 值類型不正確時，JavaScript 控制台將會顯示警告: 
    <Child name = {999} />

# 你可以讓你的 prop 只能是特定的值

  • 指定它為 枚舉類型:
Child.propTypes = {
    name : PropTypes.oneOf([value1, value2])
}
1
2
3
  • 實例:




















 








class Child extends React.Component{
    render(){
        return(
            <div>
                name: {this.props.name}
            </div>
        )
    }
}
class Parent extends React.Component{
    render(){
        return(
            <div>
                <Child name = '彪彪' />
            </div>
        )
    }
}

Child.propTypes = {
    name : PropTypes.oneOf(['牛牛', '小黑'])
}

let root = ReactDOM.createRoot(document.getElementById('root'))
root.render(<Parent />)

// 報錯: Warning: Failed prop type: Invalid prop `name` of value `彪彪` supplied to `Child`, expected one of ["牛牛","小黑"].
// 子元件的名字接受條件: 必須是 "牛牛" 或者 "小黑", 而父元件傳入的是 "彪彪"

# 整個陣列由數字組成

    Child.propTypes = {
        arr : PropTypes.arrayOf(PropTypes.number)
    }
1
2
3
  • 實例:
class Child extends React.Component{
    render(){
        return(
            <div>
                name: {this.props.arr}
            </div>
        )
    }
}
class Parent extends React.Component{
    render(){
        return(
            <div>
                <Child arr = {[1,2,3,4,5,'6']} />
            </div>
        )
    }
}

Child.propTypes = {
    arr : PropTypes.arrayOf(PropTypes.number)
}

let root = ReactDOM.createRoot(document.getElementById('root'))
root.render(<Parent />)

// 報錯: Warning: Failed prop type: Invalid prop `arr[5]` of type `string` supplied to `Child`, expected `number`.
// 因為 arr[5] == '6', 並不是一個number類型數據, 而是屬於string類型

# 默認 Prop 值

Child.defaultProps = {
    name: value
}
1
2
3
  • 實例:













 






 







class Child extends React.Component{
    render(){
        return(
            <div>
                name: {this.props.name}
            </div>
        )
    }
}
class Parent extends React.Component{
    render(){
        return(
            <div>
                <Child />
            </div>
        )
    }
}

Child.defaultProps = {
    name : '大頭大頭' // 給name屬性設置默認值
}

let root = ReactDOM.createRoot(document.getElementById('root'))
root.render(<Parent />)

// 頁面中會顯示 name: 大頭大頭, 即使父元件並沒有傳值
